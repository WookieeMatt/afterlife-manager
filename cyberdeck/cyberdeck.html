◢ HOW TO USE

QUICK START

1. Click JACKED OUT in the header to jack in. NET Actions become available.

2. Click any Interface Ability button. Set the DV and any mods. Click EXECUTE.

3. The roll appears in the Roll Console below. Click [COPY → ROLL20] to put a Roll20-formatted message in your clipboard. Paste it in Roll20 chat with Ctrl+V — your table sees the same number you saw.

PROGRAMS

Click REZ to activate a program (costs 1 NET Action). Click FIRE on Attacker programs to roll their attack vs an engaged target's defense.

Use ACTIVE CYBERDECK to switch hardware while jacked out. DECKS / SHOP (buy decks, upgrade slots/quality, duplicate/delete) is only available while jacked out — the button is disabled during a netrun.

With more than one deck, you may get a session chooser on load (unless you tick "Remember for this tab").

Use SWAP LOADOUT to move programs between the active deck and your home library; library counts are shared across all decks so you cannot load more physical copies than you own. Slot limit follows the active deck.

Black ICE and KRASH Barrier each take 2 slots.

NET COMBAT

When you trigger a Black ICE or it lies in wait — pick it from the dropdown and click + ENGAGE. Track its REZ with the −/+ buttons. When REZ hits 0, it's Derezzed.

CRITICALS

Natural 10 → roll another d10 and ADD. Natural 1 → roll another d10 and SUBTRACT. Each chains only once. Cyberdeck handles this automatically.

CHARACTER

Open ◈ PROFILE → INVENTORY to add or remove weapons, armor, gear, clothing, vehicles, and cyberware. Housing fields save when you change them. The vitals bar shows your balance next to a credit-style chip.

Character sheet (JSON): Open ⚙ CHARACTER and use EXPORT JSON to download character, skills, lifepath, defaults (handle, maxes, deck slots, default loadout), playerInventory, and when present cyberdecks plus activeCyberdeckId. IMPORT JSON loads that bundle; if the file includes playerInventory, it replaces your current inventory block. Files are your own data — not official R. Talsorian / PDF content.

The AGENT strip (Core p.352) tracks the 100eb Premium smartphone gear: tick Agent-suggested outfit when you are wearing its seasonal picks to include +2 on Library Search and Wardrobe & Style checks (rolled from PROFILE → SKILLS).

AGENT DESK & OLLAMA

Open AGENT DESK on the Agent strip for chat, optional local Ollama replies, and your own mission notes or imported .md / .txt excerpts. This app does not ship Cyberpunk RED rulebook text — only material you paste or load is sent as context.

CORS & origins: The browser blocks cross-origin requests unless Ollama allows your page’s origin. Set environment variable OLLAMA_ORIGINS before starting Ollama (e.g. include http://localhost:PORT if you serve the deck from a static server, or the exact file:// origin if you open the HTML file directly — the latter is fragile). On Windows, set the variable in System Environment Variables or your shell, then restart Ollama.

file:// vs local server: Opening cyberdeck.html as a raw file:// URL often fails to reach http://127.0.0.1:11434 until origins are configured. Serving the folder with any static HTTP server (e.g. npx serve) and adding that origin to OLLAMA_ORIGINS is usually easier.

With Ollama disabled, use Inject GM note for a visible [GM] line in the transcript (no simulated LLM). Enable Ollama, set base URL and model, then Test connection before sending chat.

NET ARCHITECTURE

Under NET COMBAT, load a markdown .md map (or pick a sample). Hidden sections sit inside netarch:gate comments in the file. Run PATHFINDER, set the modal DV to match a hidden sector’s Pathfinder DV, and beat it (total > DV) to reveal that sector. Optional floor="N" on a gate means you must be on that floor. GM reveal all bypasses locks for solo testing.

PERSISTENCE

Everything saves to your browser's localStorage automatically. Close the tab, reopen — picks up where you left off. Use RESET ALL if you want a clean slate.

When opened inside Foundry VTT (Afterlife Manager module), saves are scoped per character via the URL so different actors do not overwrite each other.
